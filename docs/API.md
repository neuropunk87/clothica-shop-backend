# Clothica API Reference

This document describes the current REST API contract. All endpoints are mounted under `/api` unless explicitly stated otherwise.

## Conventions

- Content type: `application/json`, except image uploads which use `multipart/form-data`.
- Authentication: HTTP-only cookies set by auth endpoints.
- Language detection: `Accept-Language` header or query string, supported languages are `uk` and `en`.
- IDs: MongoDB ObjectId strings.
- Dates: ISO 8601 strings where returned by MongoDB or generated by the API.

## Authentication Cookies

| Cookie | Purpose | Lifetime |
| --- | --- | --- |
| `accessToken` | Authenticates protected API requests | 15 minutes |
| `refreshToken` | Rotates an expired session | 1 day |
| `sessionId` | Identifies server-side session record | 1 day |

The database stores HMAC hashes of session tokens, not raw bearer values.

## Error Shape

Typical API errors use this shape:

```json
{
  "success": false,
  "message": "Validation error",
  "errors": []
}
```

Common status codes:

| Status | Meaning |
| --- | --- |
| `400` | Validation error or invalid request |
| `401` | Missing or invalid authentication |
| `403` | Forbidden, CSRF origin failure or missing admin role |
| `404` | Resource not found |
| `409` | Duplicate resource |
| `413` | Request body is too large |
| `429` | Rate limit exceeded |
| `500` | Unexpected server error |

## Endpoint Matrix

### Auth

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `POST` | `/auth/register` | Public | Create a user and set session cookies |
| `POST` | `/auth/login` | Public | Authenticate by phone/password and set session cookies |
| `POST` | `/auth/logout` | Authenticated | Delete current session and clear cookies |
| `POST` | `/auth/refresh` | Cookie refresh session | Rotate session cookies |
| `POST` | `/auth/request-password-reset` | Public | Send reset code via Telegram when account is linked |
| `POST` | `/auth/reset-password` | Public | Reset password with Telegram code |

Register request:

```json
{
  "name": "Olena",
  "phone": "+380991234567",
  "password": "strong-password"
}
```

Password rules:

- `password`: 10-128 characters for registration/reset.
- `phone`: Ukrainian format, `+380XXXXXXXXX`.

### Users

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `GET` | `/users/profile` | Authenticated | Get current profile |
| `PATCH` | `/users/profile` | Authenticated | Update `name`, `lastname`, `city`, `branchnum_np` |
| `DELETE` | `/users/profile` | Authenticated | Delete profile, sessions and avatar asset |
| `GET` | `/users/profile/telegram-link` | Authenticated | Generate a short-lived Telegram linking URL |

Profile update request:

```json
{
  "name": "Olena",
  "lastname": "Koval",
  "city": "Kyiv",
  "branchnum_np": "12"
}
```

Phone and email cannot be changed through the profile update endpoint.

### Categories

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `GET` | `/categories` | Public | Paginated category list |
| `GET` | `/categories/:id` | Public | Category details |
| `POST` | `/categories` | Admin | Create category |
| `PATCH` | `/categories/:id` | Admin | Update category |
| `DELETE` | `/categories/:id` | Admin | Delete category |
| `PATCH` | `/categories/:id/img` | Admin | Upload category image |

List query:

| Query | Type | Default | Constraints |
| --- | --- | --- | --- |
| `page` | integer | `1` | min `1` |
| `perPage` | integer | `6` | `3-7` |

Create category request:

```json
{
  "name": "Dresses",
  "description": "Seasonal dresses"
}
```

Image upload:

- Form field: `img`
- Max size: 2 MB
- Allowed types: JPEG, PNG, WebP, GIF

### Goods

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `GET` | `/goods` | Public | Product list with filters/sorting |
| `GET` | `/goods/:id` | Public | Product details |
| `POST` | `/goods` | Admin | Create product |
| `PATCH` | `/goods/:id` | Admin | Update product |
| `DELETE` | `/goods/:id` | Admin | Delete product |

List query:

| Query | Type | Notes |
| --- | --- | --- |
| `page` | integer | min `1`, default `1` |
| `perPage` | integer | `5-20`, default `12` |
| `gender` | string | `women`, `unisex`, `man` |
| `size` | CSV string | `XS,S,M,L,XL,XXL` |
| `good` | CSV ObjectId string | Filter by product IDs |
| `color` | CSV string | Must be in configured color list |
| `category` | ObjectId or slug | Slug is resolved before validation |
| `minPrice` | number | positive |
| `maxPrice` | number | positive |
| `search` | string | max `128` |
| `sortBy` | string | `price`, `name`, `popgoods` |
| `sortOrder` | string | `asc`, `desc` |

Create product request:

```json
{
  "name": "Linen Dress",
  "category": "65f000000000000000000000",
  "image": "https://example.com/image.webp",
  "price": {
    "value": 2499,
    "currency": "грн"
  },
  "size": ["S", "M", "L"],
  "colors": ["black"],
  "description": "Lightweight linen dress",
  "prevDescription": "New collection",
  "gender": "women",
  "characteristics": ["linen", "summer"]
}
```

### Orders

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `GET` | `/orders` | Authenticated | Current user's orders |
| `POST` | `/orders` | Public | Create order |
| `PATCH` | `/orders/:id/status` | Admin | Update order status |

Create order request:

```json
{
  "goods": [
    {
      "goodId": "65f000000000000000000000",
      "amount": 1,
      "size": "M"
    }
  ],
  "sum": 2499,
  "userName": "Olena",
  "userLastName": "Koval",
  "userPhone": "+380991234567",
  "city": "Kyiv",
  "branchnum_np": "12",
  "comment": ""
}
```

Important: `sum` remains in the request schema for frontend compatibility, but the API recalculates and stores the final value from current MongoDB product prices. Clients must not rely on being able to set totals manually.

Order status values:

```text
pending, processing, shipped, delivered, cancelled
```

### Feedbacks

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `GET` | `/feedbacks` | Public | Paginated feedback list |
| `POST` | `/feedbacks` | Public | Create guest feedback |

List query:

| Query | Type | Default | Constraints |
| --- | --- | --- | --- |
| `page` | integer | `1` | min `1` |
| `perPage` | integer | `3` | `3-12` |
| `good` | ObjectId | - | product filter |
| `category` | ObjectId | - | category filter |
| `rate` | integer | - | `1-5` |

Create feedback request:

```json
{
  "author": "Guest",
  "rate": 5,
  "comment": "Great quality",
  "good": "65f000000000000000000000",
  "category": "65f000000000000000000001"
}
```

Feedbacks are intentionally public by product requirements and are protected by validation and public write rate limiting.

### Subscriptions

| Method | Path | Access | Description |
| --- | --- | --- | --- |
| `POST` | `/subscriptions` | Public | Subscribe an email to newsletter |

Request:

```json
{
  "email": "customer@example.com"
}
```

Existing subscriptions return success without disclosing account state beyond the generic success message.

## AdminJS

AdminJS is available at `/admin`. The same users collection is used for authentication; only users with `role: "admin"` can sign in.

Admin resources:

- Users
- Goods
- Categories
- Orders
- Feedbacks
- Subscriptions

## Swagger

Swagger UI is mounted at `/api-docs` when:

- `NODE_ENV !== production`, or
- `ENABLE_SWAGGER=true`

Swagger describes the public API surface, but this Markdown reference is the canonical human-readable guide for current behavior.
